Podstawy

Konto i instalacja — wystartuj w 5 minut

Załóż konto na huggingface.co, zainstaluj biblioteki i zaloguj się przez CLI. Od tego momentu masz dostęp do setek tysięcy modeli i datasetów.

# Zainstaluj główne biblioteki
pip install transformers datasets huggingface_hub

# Zaloguj się tokenem z huggingface.co/settings/tokens
huggingface-cli login

# Sprawdź czy działa
huggingface-cli whoami

Podstawy

Hub — GitHub dla modeli AI

Hugging Face Hub to centralne repozytorium modeli, datasetów i Spaces. Każdy element ma kartę (model card) z metrykami, licencją i przykładami użycia.

Model card Zawsze sprawdź licencję (Apache, MIT, Llama, OpenRAIL), rozmiar modelu, dataset treningowy i metryki — to widać na karcie modelu.
03
Workflow

Pipeline — AI w trzech liniach kodu

pipeline() to abstrakcja, która ukrywa tokenizer, model i post-processing. Wybierasz task, dostajesz gotowy obiekt — idealne na start.

from transformers import pipeline

# Sentyment — model dobiera się automatycznie
clf = pipeline("sentiment-analysis")
clf("Uwielbiam Hugging Face!")
# [{'label': 'POSITIVE', 'score': 0.998}]

# Z konkretnym modelem
gen = pipeline("text-generation", model="gpt2")
gen("AI in 2026 will", max_length=30)
04
Podstawy

Tokenizer — tekst staje się liczbami

Modele nie rozumieją tekstu, tylko liczby. Tokenizer dzieli tekst na tokeny (sub-słowa) i mapuje je na ID. Zawsze używaj tokenizera dopasowanego do modelu.

from transformers import AutoTokenizer

tok = AutoTokenizer.from_pretrained("bert-base-uncased")
out = tok("Hugging Face is awesome", return_tensors="pt")

# input_ids: [101, 17662, 2227, 2003, 12476, 102]
# tokens: [CLS] hugging face is awesome [SEP]
# attention_mask: [1, 1, 1, 1, 1, 1]
Pro tip Tokenizer i model muszą być z tej samej rodziny. BertTokenizer + GPT2Model = błąd. Używaj AutoTokenizer + ten sam checkpoint.
05
Modele

AutoModel — jeden interfejs, każda architektura

Klasy AutoModel, AutoModelForCausalLM, AutoModelForSequenceClassification wybierają poprawną architekturę po nazwie checkpointu. Ładujesz BERT-a, Llamę i Mistral tym samym kodem.

from transformers import AutoTokenizer, AutoModelForCausalLM
import torch

ckpt = "mistralai/Mistral-7B-Instruct-v0.2"
tok = AutoTokenizer.from_pretrained(ckpt)
model = AutoModelForCausalLM.from_pretrained(
  ckpt, torch_dtype=torch.float16, device_map="auto"
)

inputs = tok("Wyjaśnij Hugging Face", return_tensors="pt")
out = model.generate(**inputs, max_new_tokens=100)
print(tok.decode(out[0]))
06
Datasets

Datasety — load_dataset() i po sprawie

Biblioteka datasets daje dostęp do setek tysięcy zbiorów w jednej linii. Streaming, lazy loading, transformacje, split train/test — wszystko gotowe.

load_dataset() .map() .filter() streaming=True .train_test_split() push_to_hub()
from datasets import load_dataset

# Pobierz dataset (cache lokalny)
ds = load_dataset("imdb")
print(ds["train"][0])
# {'text': '...', 'label': 1}

# Streaming — bez pobierania całości
ds = load_dataset("c4", "en", streaming=True)
for ex in ds["train"].take(5):
  print(ex["text"][:100])

Możesz też wgrać własny dataset: ds.push_to_hub("user/nazwa") — staje się publiczny lub prywatny w 5 sekund.

07
Spaces

Spaces — wrzuć demo, dostaniesz URL

Spaces to darmowy hosting aplikacji ML. Najpopularniejsze frameworki: Gradio (3 linijki = UI), Streamlit, czy własny Docker. Push do repo = deploy.

# app.py — kompletne demo w 5 liniach
import gradio as gr
from transformers import pipeline

clf = pipeline("sentiment-analysis")
gr.Interface(fn=clf, inputs="text", outputs="label").launch()
08
Pro

Fine-tuning — dostosuj model do swoich danych

Klasa Trainer ogarnia training loop, ewaluację, checkpointy i logowanie. Dasz dataset, model i argumenty — dostajesz wyfine-tunowany model gotowy do push_to_hub().

from transformers import (
  AutoModelForSequenceClassification, AutoTokenizer,
  TrainingArguments, Trainer
)
from datasets import load_dataset

ds = load_dataset("imdb")
tok = AutoTokenizer.from_pretrained("distilbert-base-uncased")
model = AutoModelForSequenceClassification.from_pretrained(
  "distilbert-base-uncased", num_labels=2
)

args = TrainingArguments(
  output_dir="./out", num_train_epochs=3,
  per_device_train_batch_size=16,
  push_to_hub=True
)
Trainer(model=model, args=args, train_dataset=ds["train"]).train()
Mniej GPU? Sprawdź PEFT i LoRA — fine-tuning miliardów parametrów na 16 GB VRAM. Plus bitsandbytes do kwantyzacji 4/8-bit.
09
Inference

Inference API — model w chmurze przez HTTP

Nie chcesz hostować modelu lokalnie? Hugging Face daje Inference API (serverless, do testów) i Inference Endpoints (dedykowane, produkcja). Wywołujesz przez REST.

from huggingface_hub import InferenceClient

client = InferenceClient(token="hf_xxx")

out = client.text_generation(
  "Wyjaśnij Hugging Face jednym zdaniem:",
  model="meta-llama/Meta-Llama-3-8B-Instruct",
  max_new_tokens=80
)
print(out)

Bezpieczeństwo

Tokens — uwierzytelniaj się raz i z głową

Tokeny dostępu (Access Tokens) zastępują hasło. Twórz osobne tokeny per projekt z najmniejszymi możliwymi uprawnieniami (read / write / fine-grained). Nigdy nie wrzucaj ich do repo.

# 1. Wygeneruj token: huggingface.co/settings/tokens
# → New token → fine-grained → tylko potrzebne scope'y

# 2. Zaloguj się przez CLI (zapisuje w ~/.cache/huggingface)
huggingface-cli login

# 3. Lub przez zmienną środowiskową
export HF_TOKEN="hf_xxx..."

# 4. W kodzie — nigdy nie hardkoduj!
import os
from huggingface_hub import login
login(token=os.environ["HF_TOKEN"])
